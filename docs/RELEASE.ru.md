# Руководство по релизам

[English](RELEASE.md) | [Русский](RELEASE.ru.md)

System-I2 использует GitHub Actions, чтобы собрать desktop-пакеты и загрузить их в GitHub Releases.

## Процесс релиза

Процесс описан в `.github/workflows/release.yml`.

Он запускается, когда:

- отправлен тег, подходящий под `v*`;
- процесс запущен вручную с параметром `release_tag`.

Матрица сборки запускается на:

- `macos-latest`
- `windows-latest`
- `ubuntu-22.04`

Каждый раннер устанавливает Node.js 20, стабильный Rust, зависимости интерфейса через `npm ci`, а затем запускает `tauri-apps/tauri-action@v0.6.2`, чтобы собрать и загрузить пакеты.

## Публикация v1

Создайте и отправьте релизный тег:

```bash
git tag v1
git push origin v1
```

GitHub Actions создаст или обновит релиз `v1` и прикрепит собранные desktop-пакеты как артефакты релиза.

Ссылка на релиз:

```text
https://github.com/itslaputa/System-i2-pub/releases/tag/v1
```

## Проверки безопасности перед тегом

Перед отправкой релизного тега запустите:

```bash
rg -n --hidden -e "private-path-marker" -e "local-user-path-marker" -g '!.git' .
find . -type f \( -name "*.sqlite3" -o -name "*.sqlite3-*" -o -name "runtime-bundle.json" -o -name ".env*" \) -print
git diff --check
git status --short
```

Пользовательские данные, локальная конфигурация, секреты, сгенерированные файлы и приватные пути не должны попадать в коммит.

## Ограничения подписи

Публичный процесс не настраивает:

- подпись кода macOS;
- нотариальное заверение macOS;
- подпись кода Windows;
- подпись Linux-пакетов.

Собранные пакеты подходят для приватной проверки и внутреннего распространения, но операционные системы могут показывать предупреждения для неподписанных приложений.

## Проверка первого релиза

После первого запуска процесса для `v1`:

- проверьте, что все три job из матрицы завершились;
- убедитесь, что артефакты релиза появились на странице `v1`;
- скачайте артефакт для каждой платформы из GitHub;
- по возможности проверьте первый запуск на каждой платформе;
- убедитесь, что папка данных или пользовательская конфигурация не попали в артефакты релиза.
