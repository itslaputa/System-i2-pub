# Руководство по данным приложения

[English](DATA.md) | [Русский](DATA.ru.md)

System-I2 работает по модели local-first. Пользовательские задачи, проекты, категории, комментарии и исходные данные аналитики хранятся на машине пользователя в папке данных приложения. Публичный репозиторий не должен содержать реальные пользовательские данные.

## Папка данных

Папка данных содержит четыре файла:

- `tasks.sqlite3`
- `task_categories.json`
- `task_category_change_log.log`
- `project_categories.json`

SQLite-база хранит задачи и проекты. JSON-файлы хранят категории задач и типы проектов. Файл журнала изменений хранит историю изменений категорий.

## Первый запуск

Если папка данных не настроена, приложение открывает экран настройки и предлагает три варианта:

- подключить существующую папку данных;
- создать папку данных в выбранном месте;
- создать папку данных в стандартной системной папке приложения.

Все файлы данных должны лежать в одной папке.

## Пользовательская настройка

Выбранный путь к папке данных сохраняется в `runtime-bundle.json` внутри стандартной системной папки приложения.

Отладочные сборки используют имя папки `System-I2-Dev`. Релизные сборки используют `System-I2`. Код должен получать эти пути через системные API, а не через жёстко заданные локальные пути.

> [!IMPORTANT]
> Не добавляйте приватные абсолютные пути в код, документацию, тесты или примеры конфигурации.

## Данные для разработки

В репозитории есть `Data/.gitkeep`, чтобы у разработчиков было понятное место для локальных данных во время разработки. Содержимое `Data/` игнорируется.

Чтобы сбросить локальные данные разработки:

1. Закройте приложение.
2. Удалите или переместите созданную папку данных.
3. Удалите `runtime-bundle.json` для сборки разработки из стандартной системной папки приложения.
4. Запустите приложение снова и пройдите первый запуск.

Не коммитьте файлы данных, которые появятся после этого.

## Резервные копии

В настройках есть действие для создания резервной копии данных. Оно создаёт zip-файл из активной папки данных и сохраняет его на системный рабочий стол.

Резервные копии включают содержимое активной папки данных, в том числе служебные SQLite-файлы вроде `tasks.sqlite3-wal` и `tasks.sqlite3-shm`, если они есть.

## Ручное копирование

Для ручного копирования данных закройте приложение и скопируйте всю папку данных целиком. Так все четыре файла останутся согласованными между собой.

Более удобный вариант — использовать кнопку резервного копирования в настройках, чтобы приложение записало один zip-архив.

## Что нельзя коммитить

Никогда не коммитьте:

- `tasks.sqlite3`
- `tasks.sqlite3-wal`
- `tasks.sqlite3-shm`
- `runtime-bundle.json`
- `task_categories.json`
- `task_category_change_log.log`
- `project_categories.json`
- любую скопированную системную папку данных приложения;
- любой zip-архив с пользовательскими данными.

Публичный репозиторий должен содержать только шаблоны, исходный код, тесты и документацию.
