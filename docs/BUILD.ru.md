# Руководство по сборке

[English](BUILD.md) | [Русский](BUILD.ru.md)

Это руководство описывает локальную разработку, сборку desktop-приложения и минимальные проверки перед публикацией релизного артефакта.

## Требования

- Node.js 20+
- npm
- Rust toolchain
- [предварительные требования Tauri 2](https://tauri.app/start/prerequisites/) для целевой ОС

Сначала установите зависимости интерфейса:

```bash
npm install
```

Для подготовки конкретной платформы используйте официальное руководство Tauri prerequisites для вашей ОС.

## macOS

Запуск приложения в режиме разработки:

```bash
npm run tauri dev
```

Сборка macOS `.app`:

```bash
npm run tauri build -- --bundles app
```

Готовое приложение создаётся здесь:

```text
src-tauri/target/release/bundle/macos/System-I2.app
```

Подпись и нотариальное заверение macOS в этой публичной копии не настроены. Считайте неподписанные локальные сборки артефактами для разработки, пока подпись релиза не будет добавлена и проверена.

## Windows

Установите требования Tauri для Windows, включая Rust toolchain и системные компоненты WebView для Tauri 2.

Команда разработки:

```bash
npm run tauri dev
```

Команда сборки:

```bash
npm run tauri build
```

Windows-пакеты нужно проверять на Windows перед публикацией. Не считайте непроверенную кроссплатформенную сборку готовым production-релизом.

## Linux

Установите требования Tauri для вашего дистрибутива, включая Rust toolchain и нужные системные библиотеки WebKit.

Команда разработки:

```bash
npm run tauri dev
```

Команда сборки:

```bash
npm run tauri build
```

Linux-пакеты нужно проверять на целевом дистрибутиве перед публикацией.

## Минимальная проверка релиза

Запустите проверки:

```bash
npm run test:frontend
npx tsc --noEmit --pretty false
npm run build
cargo test --manifest-path src-tauri/Cargo.toml
cargo check --manifest-path src-tauri/Cargo.toml
```

Затем проверьте поведение приложения:

- экран первого запуска появляется, когда папка данных ещё не настроена;
- создание папки данных в стандартной системной папке приложения работает;
- создание папки данных в выбранном месте работает;
- подключение существующей папки данных работает;
- экран записи задач, выбор проекта, аналитика и настройки открываются без ошибок;
- резервная копия из настроек записывает zip-файл на системный рабочий стол;
- пользовательские данные, локальная конфигурация и сгенерированные файлы не попали в репозиторий.
